## 1. Kill the "Instant Workshop" wording

The phrase is generated by the matchmaker RPCs (`join_lounge`, `join_medium_lounge`) and persisted as `instant_rooms.title`, e.g. `Instant Workshop: Film`. Rather than touch DB functions, normalize at the display layer so existing rows and any future RPC output both render correctly.

- Add a tiny helper `formatRoomTitle(title, medium)` in `src/lib/instant.ts` (new):
  - If the title matches `/^instant\s+workshop(\s*[:\-–]\s*(.*))?$/i`, rewrite to `Workshop: <Label>` using the captured medium label (or `CATEGORIES.find(c => c.id === medium)?.label` as fallback). If neither, return `Workshop`.
  - Otherwise return the original title unchanged (preserves host-set titles like `Maya's Workshop`).
- Apply in every surface that renders `room.title`:
  - `src/components/live-workshops-rail.tsx` (both card variants, lines 116 + 170)
  - `src/components/instant-activity-ticker.tsx` (ticker line)
  - `src/routes/workshop.$id.tsx` header/title and `head()` meta
  - `src/components/notifications-bell.tsx` (if it shows the title)
- Update the `head()` meta on `/workshop/$id` to `Workshop: Film — drop in` style.
- Comment cleanup: update the `Instant Workshops` references in `src/lib/mediums.ts` and `src/lib/instant.functions.ts` JSDoc to "live Workshops" so future devs don't reintroduce the term.

No migration required; no backend rename needed. Existing in-flight rooms instantly display the new convention.

## 2. Light polish pass on /workshop (within the current layout)

These are small, surgical refinements — no structural rework.

**Featured "Any topic" card (left column)**
- Replace the `START THE NIGHT` eyebrow with a live-aware label: when 0 rooms are live → `START THE NIGHT`; when ≥1 room is live → `JUMP IN` with the live count pill (`· {n} live`). Avoids the awkward "Start the night" copy at 9pm when rooms already exist.
- Tighten vertical rhythm: the column currently has a lot of dead air on desktop. Add a subtle ambient detail — three faint gradient dots forming an arc behind "Any topic" (uses existing `gradient-motion` token at low opacity). Keeps the column feeling intentional instead of empty.
- CTA copy adapts: `Open the first room →` (empty) / `Match me to a seat →` (when live rooms exist).

**Topic list (right column)**
- Replace the `Open first` chip on empty rows with a quieter `+ Start` ghost chip, and keep the prominent filled chip for `Take a seat` on live rows. This visually separates "join" (primary) from "create" (secondary) and stops the list from looking like 8 identical CTAs.
- Add a tiny inline avatar stack (max 3, `-space-x-1`) to live rows when participants exist, pulled from the same `listActiveInstantRooms` payload. Makes "live" feel populated.
- The `0 people live / N people live` summary should pluralize correctly (`1 person live`) and switch color to `text-ink` when > 0.

**Header**
- The orange dot left of the `Workshop` wordmark in the top-left nav is fine, but the `· Workshop` separator dot in the page header (`<span className="text-ink-muted/40">·</span>`) is dangling next to the back arrow. Drop it — back arrow + wordmark is enough.
- Mic/cam status dots get a single combined tooltip ("Mic & camera ready" / "No camera detected") instead of two separate `title=` attrs.

**Host strip**
- Copy refinement: `Want host controls?` → `Hosting a session?` — more inviting, less feature-spec.
- Subtitle: `Name the room, pick visibility, share the link.` → `Name it, lock it down, share a link.` — tighter, friendlier.

**Micro polish**
- The `Fewer / More` toggle at the bottom of the topic list: add a subtle hairline divider above it and reduce the chevron size to 12px so it reads as secondary chrome.
- Add `focus-visible:ring-2 ring-primary/30` to all topic rows for keyboard a11y (currently only ring-1).
- The footer fine-print line (`Everything in a live room is ephemeral…`) can move into a tooltip on a small `(i)` icon next to the live counter — frees a row of vertical space and keeps the page denser.
